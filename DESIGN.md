# Academic Research Skills — GitHub Pages Design Doc

**Approach:** Jekyll + GitHub Pages  
**Layout:** Search-first grid (Layout B)  
**Hosting:** GitHub Pages on `github.com/lerlerchan/academic_research_skills`

## Site Architecture

### Directory Structure
```
academic_research_skills/
├── _config.yml                    # Jekyll configuration
├── _data/
│   └── skills.yml                 # Single source of truth for all skills
├── _layouts/
│   ├── default.html               # Base layout (header, footer, styling)
│   ├── home.html                  # Homepage with search-first grid
│   └── skill.html                 # Individual skill detail page
├── _includes/
│   ├── search-bar.html            # Reusable search component
│   └── skill-card.html            # Reusable card component
├── assets/
│   ├── css/
│   │   └── style.css              # Custom CSS (Tailwind or vanilla)
│   └── js/
│       └── search.js              # Client-side Lunr.js filter
├── skills/
│   └── index.html                 # Permalink: /skills/
├── index.html                     # Homepage (permalink: /)
├── README.md                      # Setup instructions
└── Gemfile                        # Ruby dependencies

pages/ (auto-generated by Jekyll)
├── index.html                     # From index.html template
└── skills/[skill-id]/index.html   # From skills/index.html + Liquid loop
```

### Data Structure (`_data/skills.yml`)

Each skill has:
- **id** — URL slug (e.g., `slr-prisma`)
- **name** — Display name
- **category** — One of: Literature Review, General, Writing, Methods
- **description** — Short summary (100-150 words)
- **repository** — GitHub repo URL
- **asset_url** — Direct link to raw `.md` file or `.zip`
- **asset_type** — `md`, `md+references`, `system`
- **author** — Creator name
- **tags** — Search keywords (optional)

**Example (3 skills):**

## Search Implementation

### Strategy
1. **Lunr.js**: Lightweight full-text search
2. **Pre-indexed**: Index built during Jekyll build (static)
3. **Client-side only**: No server queries, instant results

**Setup:**
```javascript
// search.js
const lunr = require('lunr');

const documents = [
  { id: 'slr-prisma', title: 'SLR-PRISMA', body: '...', category: 'Literature Review' },
  // ... (6 skills)
];

const idx = lunr(function() {
  this.ref('id');
  this.field('title', { boost: 10 });
  this.field('body', { boost: 5 });
  this.field('category');
  
  documents.forEach(doc => this.add(doc));
});

// On page load, bind search input
document.querySelector('#search').addEventListener('keyup', (e) => {
  const query = e.target.value;
  const results = idx.search(query);
  renderResults(results);
});
```

## Styling

- **Primary**: Purple (`#7c3aed`)
- **Background**: White (`#ffffff`)

```yaml
  - .git
  - .gitignore
  - Gemfile
  - Gemfile.lock
```

### `Gemfile`
```ruby
source "https://rubygems.org"
gem "jekyll", "~> 4.3"
gem "jekyll-sitemap"
gem "jekyll-feed"
gem "webrick", "~> 1.8"
```

### GitHub Actions Workflow (`.github/workflows/jekyll.yml`)
- Trigger: Push to `main`
- Steps:
  1. Checkout code
  2. Setup Ruby + bundler
  3. Install gems
  4. Run `bundle exec jekyll build`
  5. Deploy to `gh-pages` branch
- Auto-deploys to GitHub Pages
